An introduction into elements

HTML becomes fairly basic once you understand how it works.

A HTML page is made up of many elements which form together to make a web page which a browser can display.

Each element consists of a starting tag, content and then an ending tag. The starting tag has a unique set of letters that stand for a specific word or sometimes multiple words inside chevrons or <> (there is a link to a full list of tags further down the page). You should also try and write tags in lower case as they look neater, are easier to write and some html octypes require lowecase elements. An example of this is <p> which stands for paragrapgh and tells the browser that you are beginning a paragrapgh element.

After you have opened a tag you usually want to put some text or content into it. You do this by just typing text between the opening and closing tag which you will now create. You close a basic tag by just repeating the opening tag except with a forward slash before the letters like this </p>.

A completed element looks something like this <p>Content</p>, you can also see this in the image below.

Tags without content

There are some tags without content and these tags sometimes link to something or add style to a page. Examples of these are <img> tags which add images, <link> tags which help link to external style sheets and <br> or break tags which force a line break.

Note: These tags have to be self closed even though they will work without doing this. To self close contentless tags you put a forward slash just before the end chevron like this: <img/>. Be warned the tag to the left is not complete as it is missing attributes.

What are attributes?

Attributes usually provide additional information to a tag however some tags like the img, link and a tag will not work without them. Attributes are always written in the start tag and should be written in lower case.

Different attributes specify different things and some attributes can only be used by 1 or 2 tags, however there are universal attributes like class, id and style which all have something to do with style and will be talked about later when you venture into CSS territory.

As you can see attributes have spaces either side of them. Attributes also have are created in and value pairs and are separated by a equals sign. The value also has to be inside of quatation marks.

General text

For general text you can either use the <p> tag or the <div> tag. For our website we are going to be using the <p> tag (which stands for paragragh) as it has preset padding that we are going to use. A <div> tag has no preset padding and won't look good on our page without styling. However these tags are also very useful for dividing parts of your page and having a tag with no style. A webpage will usually display text if it is not inside one of these tags, however your page will not validate later.

Validating

Now that you have a website your going to want to validate it so that you can be sure it is going to work with all web browsers. Validating is important as it means that even tbough your website might work fine now it might not work in the future due to changes in the web browsers.

Validating also means that your code requires less maintenance so if you can validate now, you are going to save yourself a lot of time in the future. Furthermore validating is a sign of professionalism and it also helps you check your work for debugging incase your website doesn't work and you don't know what is wrong.

Now as you may or may not know there are different kinds of html and some have different features and requirements for validating. These different types of html are specified in the doctype tag which always goes in the top line of a html page. You are learning XHTML 1.0 Strict as this is probably the easiest html type to learn and has extra requirements that teach you good practices.

The doctype for XHTML 1.0 Strict is (copy and paste it into :

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.0 Strict//EN" "http://www.w3.org/TR/xhtml1/DTD/xhtml1-strict.dtd">

We still got to do 2 more things until your page validates:

1. Add this attribute to your html tag: xmlns="http://www.w3.org/1999/xhtml" lang="en"

2. Copy and paste this meta tag into the head tag: <meta http-equiv="content-type" content="text/html; charset=UTF-8"/>

Creating tables

Tables are easy to make when you know how to make them. There are 4 tags you need to know to know how to create a table you can see these in the table below:

Tag What it is used for
table All the elemnts that make up a table lye inside of the table tag.
tr The tr tag stands for table row and specifies a new table row. Table rowss automatically stack vertically to create rows inside a table, however this tag needs a td or th tag to be visible.
td This tag stands for table data and creates a cell inside a table row. Cells stack horizontally to create columns and you should have an equal amount of td tags in a tr tag to make the table look even.
th This tag stands for table heading and just like the td tag it creates a cell inside a table row. The only difference between these two tags is that the th tag has preset boldness that can be used for table headings with little work.

From reading the table above you might have understood that tables are created through individual cells and you have to have to have the same amount of cells in each row to make your table look equal. NOTE: By default the tables have no border but you can change this with the border attribute and the value of "1".
